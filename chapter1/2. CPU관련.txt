프로그램과 프로세스의 차이 :
프로그램(애플리케이션) ->실행 -> 메모리위로 올라감(프로세스) -> 프로세스는 cpu에 의해 실행됨.(스케줄링)
- 스케줄링 : 어떤 프로세스가 실행될지 결정하는 것

- 애플리케이션 실행 순서 -
하드디스크(프로그램) -> 메모리(프로세스) -> cpu

의문점 : 메모리인 프로세스 없이 하드디스크에서 전부 계산하면 되지않나??
이유 : 속도차이 때문이다. cpu -> 메모리 -> 하드디스크 순의 속도차이가 있다.
아무리 빠른 cpu의 속도를 가지더라도 하드디스크에서 전부 계산하게 되면 cpu의 속도는 하드디스크의 속도와 같아진다. (병목현상)
그렇기 때문에 메모리에 올려서 cpu에서 실행하게 된다면 메모리의 속도가 되기 때문이다.

cpu를 많이 사용하는 애플리케이션 : I/O Bound를 적게 사용하고 CPU Bound를 많이 사용하는 애플리케이션
- Hash 연산을 많이 반복하는 애플리케이션이 cpu를 많이 사용한다. (MD5 Hash연산)
