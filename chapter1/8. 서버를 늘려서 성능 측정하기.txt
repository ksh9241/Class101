성능 테스트 
500 Err : Cpu bound Application에서 자기 자신의 성능이 더 이상 이 요청에 대한 응답을 할 수 없는 상태가 되어서 cpu bound application 자체에서 떨어진 에러

502 Err : cpu bound application이 계속 던지다가 어느 순간에는 더 이상 500에러를 던질 힘 조차 없을 때 애플리케이션 자체가 종료되는 것(Hang 상태)이 502 에러이다.(성능테스트에서 이유)

502 Err 원래의미 : Gateway <-> Application 연결 끊김.

무중단 배포 방법
- docker container 연결 종료하기
- jenkins dashboard로 가서 build now를 클릭하면 무중단 배포 완료.

- docker conatiner 연결 종료하는 방법
1. docker ps 명령어 입력 (컨테이너의 대한 정보 출력 됨.)
2. docker ps |(역슬래쉬) grep {IMAGE NAMES} (우리가 선택한 image names는 빨간색으로 표시)
3. docker container kill -s 15 {container Id} 
(-s 15 : 이 컨테이너가 스스로 Graceful하게 종료할 수 있도록 해준다.)
(Graceful : 애플리케이션이 스스로 실행 중인 로직을 전부 다 처리한 다음에 종료하도록 해준다.)
