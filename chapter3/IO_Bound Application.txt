I/O 버스트 : 파일 읽기 쓰기 등을 하여 하드디스크에 접근하기 위해 대기하는 시간
I/O Bound Application이란? : I/O 버스트가 많은 것
* I/O Bound Application도 스케일 아웃이 가능하다. 단 File I/O Bound Application은 가능하다 DB I/O Bound Application은 DB의 성능을 올리지 않는 한 불가능하다.(DB에 의존적이기 때문이다.)

I/O Bound Application Scale Out경우 서버끼리 각자의 데이터를 실시간으로 공유할 수 없다.
그렇기 때문에 우리는 공유가 가능한 저장소를 사용한다. (DataBase)

I/O Bound Application의 스케일 아웃 시 문제점 : 전체 서비스의 성능을 높여도 DB의 성능이 좋지 않을 경우 전체 서비스의 성능은 DB의 성능과 동일해진다. (병목현상)

병목현상 : 두개 이상의 서비스 중 성능이 안좋은 쪽에 맞춰지는 현상
이러한 병목현상을 줄이는 방법 : 사용자의 요구데이터를 다이렉트로 DB에서 처리하지 말고 사용자의 요청을 빈 공간에 저장해두고 DB는 그 빈 공간의 요청을 처리한다.
