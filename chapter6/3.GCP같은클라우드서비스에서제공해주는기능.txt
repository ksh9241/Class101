- GCP 기준
제품 및 리소스 검색창에 *특정 키워드 검색 시 marketplace에서 image에 대한 정보가 나온다.
실행 클릭 시 vm인스턴스를 생성할 때 나오는 머신, 디스크 사이즈, 네트워크 인터페이스 등등 세팅하는 화면이 나온다.
세팅 이후 배포를 누를 경우 인스턴스가 생성된다.

*특정 키워드 : jenkins, nginx, rabbitMQ, kafka, elasticsearch 등등..

GCP에서 제공해주는 관리형 쿠버네티스 : GKE (google Kubernetes Engins)
AWS에서 제공해주는 관리형 쿠버네티스 : EKS
Azure에서 제공해주는 관리형 쿠버네티스 : AKS

nginx 생성 시 인스턴스 그룹 생성 가능
- Stateless 관리형 인스턴스 그룹 만들기 : 상태가없는 인스턴스를 만들 수 있는데 상태가 없다는 말은 데이터가 저장/보존 되지 않는다. 트래픽을 받고 처리하는 역할만 하는 인스턴스.

- Stateful 관리형 인스턴스 그룹 만들기 : 데이터베이스와 같은 스테이트 풀 워크로드용

- 새 비관리형 인스턴스 그룹 만들기 : 부하분산 (nginx와 같은 기능)

- 관리형 DB도 만들 수 있다. (mysql, postgreSQL, SQLServer 존재)
